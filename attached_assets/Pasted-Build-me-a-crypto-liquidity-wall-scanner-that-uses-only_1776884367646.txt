Build me a crypto liquidity wall scanner that uses only free public data sources. Do not rely on just one exchange. Use the best free sources available and combine them to give me the clearest possible view of where large buy orders and sell orders are sitting.

Main goal:
I want this app to scan order books and tell me where the biggest nearby buy walls and sell walls are on a coin. I do not want a complicated dashboard-first app. I want a practical scanner that returns simple outputs I can actually use.

Free data sources to use:
- Binance public market data where available
- Bybit public order book WebSocket / REST where available
- OKX public order book WebSocket / REST where available
- Kraken public order book feed where available
- Coinbase Exchange public level2 order book feed where available
- If one source is unavailable for a coin, fall back to the others
- Create a unified market data layer so the scanner can pull from whichever free exchange has that coin and enough liquidity

Important:
Use public market data only. No paid APIs. No private keys needed for scanning. WebSockets should be preferred for live market data where possible because they are better for real-time order book monitoring. Coinbase level2 is useful because it is designed for keeping a synced order book. Bybit also recommends WebSockets for market data and their public order book endpoints cover spot and derivatives. OKX also offers free REST and WebSocket market data. Kraken has public market data feeds too. 
Reference this behavior in the build:
- Coinbase level2 channel is designed for maintaining a synced order book
- Bybit public order book provides snapshot and delta logic
- OKX offers free REST and WebSocket API market data
- Kraken has public order book feeds

Core features:
- Let me enter a ticker or coin symbol and scan it
- Scan multiple exchanges for that coin if available
- Find the largest buy walls below current price
- Find the largest sell walls above current price
- Show the exact price levels where they are sitting
- Show the size of each wall in native coin size and estimated USD size
- Show how far each wall is from current price in percentage
- Rank the most important nearby liquidity zones
- Detect whether a wall is stable, getting filled, or disappearing
- Detect if a wall appears briefly and gets removed fast, and flag it as possible spoofing
- Give a simple order book pressure score from bullish to bearish
- Prefer walls within a configurable range such as 0.25%, 0.5%, 1%, 2%, 3% from price

Output style:
I want the main output to be simple and useful, not geeky.
For each coin, return:

1. Current price
2. Biggest nearby buy wall
3. Biggest nearby sell wall
4. Top 3 buy support zones
5. Top 3 sell resistance zones
6. Whether buy pressure or sell pressure is stronger
7. Whether any big wall is being pulled or eaten
8. A short AI summary paragraph in plain English

Example output:
BTCUSDT
Current price: 84,250
Biggest buy wall: 83,900 with $2.4M resting bids
Biggest sell wall: 84,900 with $1.8M resting asks
Pressure: Slightly bullish
Spoof risk: Large sell wall at 85,000 appeared then weakened
AI summary:
There is strong buy support sitting just below current price around 83,900, which may act as a bounce zone if price dips. The nearest heavy sell liquidity is around 84,900, so that is the main area to watch if price continues higher. Buyers currently look slightly stronger, but if the 83,900 wall starts getting pulled, downside could open up quickly.

AI paragraph rules:
- Keep it short and human
- Explain the important areas to watch
- Mention where buyers are defending
- Mention where sellers are stacked
- Mention whether a wall is strong, weakening, or suspicious
- Mention what area matters most next
- Do not overcomplicate the language
- Make it sound like a smart trader explaining it simply
