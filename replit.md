# TradeHub Markets — Strategy Builder Platform

## Overview
TradeHub Markets is a web platform designed for users to build, test, and automate crypto trading strategies without requiring any coding knowledge. Users can create strategies using a 7-step wizard or an AI chat builder, paper test them for free, and then deploy them live or sell them in a marketplace. The platform operates on a Free vs. Pro ($50/month) subscription model. A companion Telegram bot provides strategy trade alerts and user login UIDs. The business vision is to empower traders with accessible, sophisticated tools, expanding market potential by democratizing strategy creation and automation in the crypto space.

## System Architecture

### Core Components
- **Strategy Portal**: The main web application for strategy creation (wizard and AI chat builder), backtesting, strategy execution, and a marketplace. The wizard / AI-chat FVG condition (`strategy_ta.eval_fvg`) now supports ICT-style ATR-aware quality filters: `min_gap_atr_mult` (volatility-aware width, e.g. 2.5 = "FVG ≥ 2.5×ATR(14)"), `disp_atr_mult` (formation candle body must be ≥ N×ATR — true ICT displacement), `min_gap_usd` (absolute USD floor), `only_unfilled` (drop already-mitigated gaps), `max_age_bars` (freshness cap). Two new sub-conditions: `just_formed` (FVG completed in the last 1-2 bars — fires on the strong-displacement bar that left the gap) and `tap_and_reject` (last bar wicked into the gap and closed outside in the FVG's directional bias). The wizard FVG chip exposes all of these as Trigger chips, sliders, and a yes/no toggle; `buildWizardDescription()` carries every selection into the AI compiler description. `only_unfilled` accepts both real bools (JSON path) and strings ('true'/'false'/'yes'/'no'/'on'/'off') from the chip helper. All new fields default to 0/False so existing wizard strategies that only pass `{direction, condition, min_gap_pct}` keep their original "any 3-candle gap counts" behaviour. The AI compiler (`strategy_builder.CONDITION_SCHEMA`) documents every field and now maps natural-language phrases like "FVG ≥ 2.5x ATR", "ICT displacement creating FVG", "FVG just formed", and "tap into FVG and reject" onto the right config.
- **Telegram Bot**: A minimal companion app for trade notifications, displaying user UIDs, and an on-demand liquidity wall scanner (`/walls`).
- **Trade Tracker**: A dashboard for monitoring trade performance.
- **Day Trading Page (`/trade`)**: A public, no-authentication chart with live order block overlays, ICT Fair-Value-Gap (FVG) overlay (3-candle imbalance bands, bull/bear, unfilled-only by default — toggled via "FVG zones" in the chart toolbar with a live count badge `Nb↑ Ns↓`, polled every 30 s from `/api/trade/fvg/{symbol}` with a 25 s server cache; bands are right-anchored to the formation candle, ATR-quality-weighted opacity, and overlay an ICT consequent-encroachment 50% midline). The detector applies two volatility-aware quality filters by default: gap width must be ≥ 0.10 × ATR(14) and the formation candle's body must be ≥ 0.5 × ATR(14) (ICT "displacement"), so the same setting surfaces meaningful gaps across symbols/timeframes/regimes. Also: indicator engine (SMA/EMA/RSI/MACD/BB/VWAP/ATR/StochRSI/SuperTrend), Pine script expression evaluator, and server-evaluated chart alerts delivered via Telegram (now including a `fvg_retest` alert kind with `bull`/`bear` conditions).
- **Auto Trader (Pro)**: Allows users to save chart setups as background paper-trading strategies with AI mode (re-runs AI trade reads) or Rules mode (compiles visible indicators into a DSL — including a `fvg_retest` rule that fires when the latest bar wicks back into an unfilled FVG, taking priority over MA/MACD/etc. when the FVG toggle is on; the rule also supports a `instant_entry_atr_mult` shortcut — default 4.0 — that fires immediately in the displacement direction when a fresh FVG (≤2 bars old) is ≥ that × ATR wide, so parabolic moves aren't missed waiting for a polite retest). The Auto Trader modal exposes an "FVG entry settings" subsection (instant-entry × ATR threshold, instant-entry max age in bars, min gap × ATR, **min gap $ (absolute USD floor — e.g. set 100 to require ≥ $100 inside the FVG)**, max gap age, and a long-only/short-only/both direction filter); when the FVG toggle is on, `chartContextForSave` injects an `{type:'fvg', params:{...}}` indicator spec built from those inputs so `compile_rules` honours the user's custom thresholds + side instead of the global defaults. The side filter is enforced at evaluate-rules time (rejects opposite-side signals from `_fvg_retest_signal`) and the live "Compiled rule" preview reflects the user's settings as they tune them. A separate **"Strategy modifiers"** subsection exposes per-strategy floors saved on `chart_state.mods` and lifted to top-level `rules_json` by `compile_rules`: `min_rr_ratio` (lifts TP1 to ≥ ratio × stop distance in `derive_stop_tp_from_chart`, with TP2 preserved at 1.5× new TP1), `min_stop_pct` (widens — never tightens — the stop so distance is at least N% of entry; merged with the existing 0.3% walls floor), and `trend_ema_period` (rejects signals fighting the EMA — longs require price ≥ EMA, shorts require price ≤ EMA, evaluated in `evaluate_rules` after side filter). Each modifier is purely opt-in (zero/blank disables) so existing strategies behave identically. Each strategy supports per-strategy risk caps (max-concurrent open + daily-loss limit), risk-% position sizing (size = (equity × risk%) ÷ |entry−stop|), partial TP1 + auto-breakeven, UTC session filter (with overnight wrap; invalid HH:MM is now rejected at the API level instead of silently storing null and trading 24/7), and a cooldown after N consecutive losses. Rule-mode TPs/SLs are derived in `derive_stop_tp_from_chart` with a minimum TP1 distance floor (`MIN_TP1_PCT`, default 0.5% of entry, env-overridable) so low-timeframe trades aren't opened with sub-0.1% targets that real-world fees and spread eat alive — applies to both `walls` and `atr` source modes, with TP2 distance preserved at ≥ 1.5× TP1. AI mode is rate-limited by `AI_TICK_FLOOR_S` (default 90s, env-overridable); when Anthropic returns a credit-balance error the strategy auto-pauses for 24h. Trade-close DMs report the EFFECTIVE blended move (back-derived from total $ P&L over original notional) instead of just the final leg's raw price move — so a partial-TP1+BE-stop close that nets +0.14% no longer prints a misleading "+0.01%" — and add a "Legs:" line breaking out the TP1 partial vs runner P&L when partials fired. The Auto Trader modal also includes a per-strategy performance dashboard (equity curve, drawdown, by-hour heatmap, by-side win-rate, R-multiple) and a Rules-mode backtest replay that walks forward over the last N MEXC candles with no lookahead.
- **Built-in Backtester**: Integrated into the strategy wizard for replaying strategies against historical OHLCV data to evaluate performance. The /trade Auto Trader modal additionally exposes its own backtest endpoint (`POST /api/trade/auto/{sid}/backtest`) for Rules-mode strategies — AI-mode strategies are skipped to avoid an LLM call per bar.
- **/trade page sidebars**: Multi-symbol watchlist (top 8 perp tickers, refreshed every 12s) and a Coinglass funding/OI badge (refreshed every 60s, cached 60s) overlay the chart. A position-size calculator and a one-click alert-template picker are embedded in the corresponding modals. Persistent chart drawings have a backend (`TradeDrawing` model + `/api/trade/drawings` GET/POST/DELETE) ready for a future drawing palette UI.
- **Strategy Creator Marketplace**: Enables users to publish, sell, and purchase trading strategies.
- **Subscription Tiers**: Free tier for basic strategy building and paper testing; Pro tier for advanced features like AI Chat Builder, backtesting, live automation, and marketplace copy.
- **Public Marketing Website**: Landing page (`/`) with features, marketplace preview, and leaderboard.
- **Web Authentication System**: Supports Google OAuth and email/password registration with HMAC-signed session cookies.

### Technical Implementations
- **Database**: Neon PostgreSQL, shared across all workflows, using SQLAlchemy ORM.
- **Configuration**: Environment variables managed with `pydantic-settings`.
- **Security**: Fernet encryption, HMAC-SHA256, bearer token verification, and PostgreSQL advisory locks.
- **Risk Management**: Includes percentage-based SL/TP, risk-based position sizing, daily loss limits, and max drawdown protection.
- **AI Integration**: Hybrid approach using Gemini 2.5 Flash for scanning and Claude Sonnet 4.5 for final approval and trade review.
- **Market Analysis**: Incorporates advanced techniques like Volume Profile Analysis, Order Flow Imbalance Detection, and Chart-Based TP/SL Optimization.
- **Signal Generation**: Priority-based scanning for various trade setups (PURE TA LONGS, VWAP BOUNCE SCALPS, PARABOLIC exhausted dumps, NORMAL SHORTS, RELIEF BOUNCE longs).
- **Automated Systems**: AI Chat Assistant for recommendations, AI Trade Reviewer, and an Automatic Market Regime Detector.
- **Real-time Data**: Primarily uses MEXC for market data due to Binance geoblocking on Replit.

### UI/UX Decisions
