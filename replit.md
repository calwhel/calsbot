# TradeHub Markets ‚Äî Strategy Builder Platform

## Overview
TradeHub Markets is a web platform designed for users to build, test, and automate crypto trading strategies without requiring any coding knowledge. Users can create strategies using a 7-step wizard or an AI chat builder, paper test them for free, and then deploy them live or sell them in a marketplace. The platform operates on a Free vs. Pro ($50/month) subscription model. A companion Telegram bot provides strategy trade alerts and user login UIDs. The business vision is to empower traders with accessible, sophisticated tools, expanding market potential by democratizing strategy creation and automation in the crypto space.

### Core Components

- **Gold pip convention fixed: $0.01 ‚Üí $0.10 per pip (June 2026)**: Direct response to "wow these SL are quick" ‚Äî live XAUUSD trades stopping out almost instantly. Root cause: gold pip was defined as $0.01 (smallest price increment) but the broker/retail standard is **$0.10/pip** (a $2.45 move ‚âà 25 pips). So a user's "25 pip" gold stop became $0.25 ‚âà one full gold spread ‚Üí near-instant stop-outs, and pip P&L over-reported 10√ó. An earlier session had set gold to 0.01 to make the entry vs close cards agree, but that fixed the display mismatch in the wrong direction (down to $0.01 instead of up to the broker's $0.10). Fix: gold pip 0.01‚Üí0.10 in lockstep across the canonical sites ‚Äî `forex_engine._METAL_PIP_SIZES` (0.10) + `TYPICAL_SPREADS_PIPS` XAUUSD (25.0‚Üí2.5, real ~$0.25 spread √∑ $0.10), `ctrader_client._PIP_SIZES` (added XAUUSD:0.10, was missing ‚Üí defaulting to 0.0001) + per-symbol `pip_value` for XAU ($1‚Üí$10/pip/lot so risk-based `lots=risk/(sl_pips√ópip_value)` keeps $ risk per trade invariant), `strategy_executor._pip_size` (close/result card), and web `strategy_portal.html` (`_pipSz`, live-forex pair list, lot calculator `pipVal`, lot/paper hints). **Effect**: every existing gold strategy's stop auto-widens ~10√ó to the size the user actually intended (no longer knocked out by the spread); risk-%-sized trades keep the same $ risk (fewer/wider = correct lots); fixed-lot trades now risk ~10√ó more $ per trade than the old too-tight stop did (correct ‚Äî old was wrong). **Historical exception**: `strategy_portal_server.py::_backfill_pips` SQL deliberately kept at 0.01 (only fills `pips_pnl IS NULL` legacy pre-column rows ‚Üí keeps old gold rows consistent with their era; do NOT align to 0.10). Mobile only emits pip *counts* (backend converts) ‚Üí no mobile change. Scoped to gold; silver/platinum left at digits-based values. See `.agents/memory/metal-pip-size-drift.md`. Verified: py_compile (4 backend), mobile tsc clean, portal HTTP 200, architect review (caught a stale "$1/pip" lot hint ‚Üí fixed; re-confirmed lockstep + sizing math). Executor is prod-only/disabled in dev ‚Üí user must REPUBLISH for prod to pick it up.

- **Ghost-cleanup false-cancel of live forex trades ‚Äî THE gold-SL-no-notification root cause (June 2026)**: After shipping the forex close-reconcile (below), a real live XAUUSD short (`strategy_executions` id 5333, valid `ctrader_order_id`, `is_paper=false`) was found in Neon marked `outcome='CANCELLED'` with note "Auto-cancelled: no Bitunix order_id (ghost execution)". Root cause: the periodic ghost-cleanup sweep `_do_cancel_ghost_executions_sync` in `strategy_portal_server.py` flips any live OPEN execution with `bitunix_order_id IS NULL` (older than 5 min) to CANCELLED ‚Äî but Bitunix is the CRYPTO broker; forex/index trades are placed via cTrader and carry only `ctrader_order_id` (their `bitunix_order_id` is always NULL). So genuine live forex stop-outs were silently flipped to CANCELLED, removed from OPEN state ‚Üí nothing watched the SL fill, no win/loss notification, and the close-reconcile (which only inspects OPEN rows) couldn't help. Fix: added `AND ctrader_order_id IS NULL` to the sweep WHERE clause so only true ghosts (no broker id of ANY kind) are cancelled; updated docstring/log wording Bitunix‚Üíbroker. Recovery for already-mis-cancelled rows exists: `backfill_ghost_cancelled_executions` (re-evaluates as paper vs historical candles). NOTE: app data lives in Neon (`NEON_DATABASE_URL`), the single source of truth for all envs ‚Äî the `executeSql` replit_database target hits a stale unused DB; inspect real data via `app.database.SessionLocal`. See `.agents/memory/ghost-cleanup-broker-id-trap.md` + `app-db-is-neon.md`. Verified: py_compile, architect PASS (residual genuine-ghost edge cases bounded by the 48h stale-open sweep). Republish required for prod to pick it up.
