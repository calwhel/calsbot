# TradeHub Markets ŌĆö Strategy Builder Platform

## Overview
TradeHub Markets is a web platform designed for users to build, test, and automate crypto trading strategies without requiring any coding knowledge. Users can create strategies using a 7-step wizard or an AI chat builder, paper test them for free, and then deploy them live or sell them in a marketplace. The platform operates on a Free vs. Pro ($50/month) subscription model. A companion Telegram bot provides strategy trade alerts and user login UIDs. The business vision is to empower traders with accessible, sophisticated tools, expanding market potential by democratizing strategy creation and automation in the crypto space.

### Core Components

- **AI Strategy Tuner v3.0 ŌĆö trade-replay mode using real fills (May 2026)**: Major upgrade triggered by user feedback that v2's synthetic backtests kept reporting "0 closed trades" ŌåÆ "already well-tuned" because the backtest engine couldn't fire signals for many strategies (especially those using condition-only signals like MACD/BB/Supertrend where the engine hardcodes all params). v3 adds a **trade replay mode** that runs FIRST when the strategy has Ōēź5 closed `StrategyExecution` rows in the last 120 days. Implementation in `strategy_portal_server.py`: `_run_trade_replay_optimizer(base_config, executions, sid)` pulls each execution's `entry_price`/`direction`/`fired_at` (snapshotted into ad-hoc objects so we can close the DB session before async work), uses `app.services.strategy_executor._fetch_candles_since_entry` to fetch 1m OHLC from MEXC/Binance for each entry **once** (concurrent semaphore=6, capped at 14d window per trade, fall back to backtest if <5 trades have fetchable candles), then for every variant in `_generate_replay_variants` runs `_replay_trade(entry, dir, candles, tp%, sl%, lev)` which walks bar-by-bar checking highŌēźTP and lowŌēżSL (with industry-standard conservative tie-break: if both hit in the same bar assume SL fired first; if window expires use mark-to-market PnL on last close). `_aggregate_replay` rolls outcomes into the same stats dict shape as `run_backtest()` (total_pnl, win_rate, max_drawdown via equity-curve walk, profit_factor, liquidations as proxy of pnlŌēż-80%). Variant set for replay is wider than backtest mode (4├Ś4 SL/TP grid + 4 named combos like "Better risk/reward" + 4 leverage tweaks) since each variant is essentially free once candles are fetched ŌĆö no per-variant API calls. Endpoint wraps `_run_trade_replay_optimizer` in `asyncio.wait_for(90s)` and on timeout/None falls through to v2's backtest path. Response gains `mode: "replay"|"backtest"`, `trades_replayed`, `trades_total` fields. Mobile shows a green-bordered "Replayed N of your real trades" badge and changes section header to "N exit variants tested" in replay mode. Backtest fallback path retains all v2 functionality (primary signal sweeps for rsi/ema/sma/donchian/cci/mfi/roc/volume_spike/price_momentum/breakout, baseline_zero_trades + primary_tunable diagnostics). Concurrency 5, per-job 50s, outer = ceil(N/5)*50+20 in backtest mode.
